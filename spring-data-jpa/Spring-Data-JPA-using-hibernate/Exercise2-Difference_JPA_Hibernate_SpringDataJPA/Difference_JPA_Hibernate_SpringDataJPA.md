#### **Difference between JPA, Hibernate and Spring Data JPA :=>**



###### 1\. JPA

\- JPA stands for Java Persistence API.

\- It is a specification for object-relational mapping.

\- It is not an implementation.

\- Hibernate is one implementation of JPA.



###### 2\. Hibernate

\- Hibernate is an ORM framework.

\- It implements the JPA specification.

\- It maps Java objects to database tables.

\- It provides features like HQL, caching, and lazy loading.



###### 3\. Spring Data JPA

\- Spring Data JPA is built on top of JPA.

\- It reduces boilerplate code.

\- It provides built-in CRUD operations through repositories.

\- It integrates easily with Spring Boot.







###### Comparison



JPA

\- Specification

\- Standard API



Hibernate

\- Framework

\- JPA implementation



Spring Data JPA

\- Spring module

\- Simplifies database access



###### Conclusion



JPA defines the standard.

Hibernate implements the standard.

Spring Data JPA makes database programming easier by reducing code.
